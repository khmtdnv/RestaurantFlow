# Domain

Этот слой инкапсулирует фундаментальные бизнес-правила микросервиса. Он полностью автономен, независим от фреймворков, баз данных и протоколов передачи данных.

## Основное правило
Если бизнес-логику можно описать словами, не упоминания технические детали (БД, HTTP, кэш) — эта логика должна находиться здесь.

## Структура директорий
* `/aggregates/` — Корень агрегации (Aggregate Root). Кластер связанных сущностей и объектов-значений, которые рассматриваются как единое целое для изменения данных. Только агрегат имеет право изменять свое внутреннее состояние, гарантируя консистентность (инварианты).
* `/entities/` — Сущности (Entities). Объекты, имеющие уникальный идентификатор (ID) и изменяемое состояние на протяжении своего жизненного цикла.
* `/value_objects/` — Объекты-значения (Value Objects). Неизменяемые (Immutable) объекты без ID, описывающие характеристики (например, Цена, Адрес). Два объекта равны, если равны их атрибуты.
* `/interfaces/` — Абстрактные контракты (ABC порты). Описание интерфейсов для репозиториев и паттерна `UnitOfWork`.
* `/exceptions/` — Кастомные классы исключений доменного уровня, наследуемые от базового `DomainError`. Отражают нарушения бизнес-правил (например, `ItemUnavailableError`).

## Ограничения и зависимости
* **РАЗРЕШЕНЫ** только стандартные библиотеки Python. Использование `pydantic` допускается исключительно для валидации сложных Value Objects.
* **СТРОГО ЗАПРЕЩЕНЫ** импорты из слоев `application`, `infrastructure`, `presentation`. Слой Domain не имеет внешних зависимостей.
