# Application

Слой оркестрации, определяющий варианты использования системы (Use Cases). Он управляет потоком данных, координируя работу доменных объектов и инфраструктурных абстракций.

## Основное правило
Use Case не принимает доменных решений, он управляет процессом: принимает запрос -> валидирует права -> запрашивает данные через абстракции -> вызывает методы доменных объектов -> фиксирует транзакцию -> возвращает ответ.

## Структура директорий
* `/use_cases/` — Классы, реализующие конкретные бизнес-процессы (например, `CreateOrderUseCase`).
  * Один класс/файл соответствует одной бизнес-транзакции.
  * Взаимодействие с внешним миром происходит исключительно через инъекцию зависимостей (Dependency Injection) по интерфейсам (`IUnitOfWork`, `IRepository`).
  * Единственной публичной точкой входа является метод `execute()`.
* `/dtos/` — Объекты передачи данных (Data Transfer Objects). Pydantic-модели, используемые для строгой типизации входящих запросов (Input DTO) и исходящих ответов (Output DTO). Служат мостом между слоем Presentation и Application.

## Ограничения и зависимости
* **РАЗРЕШЕНЫ** импорты из слоя `domain`.
* **СТРОГО ЗАПРЕЩЕНА** прямая работа с инфраструктурой (например, `AsyncSession`, `Redis`, HTTP-клиенты). Взаимодействие только через интерфейсы из папки `domain/interfaces`.
* **СТРОГО ЗАПРЕЩЕНЫ** импорты из слоев `infrastructure` и `presentation`.
