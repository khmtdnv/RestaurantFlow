# Presentation

Фасад приложения. Слой отвечает за прием запросов от внешних потребителей (клиентов, других микросервисов), маршрутизацию и формирование корректных ответов (HTTP Response, AMQP Ack/Nack).

## Основное правило
Реализует принцип "Тонкого контроллера". Зона ответственности слоя: десериализация ввода -> вызов соответствующего Use Case -> возврат стандартизированного ответа.

## Структура директорий
* `/api/` — HTTP-интерфейс (FastAPI роутеры).
  * `/v1/` — Версионированные контроллеры, сгруппированные по доменным сущностям (`cart.py`, `order.py`).
  * `router.py` — Корневой агрегатор всех HTTP-маршрутов.
* `/amqp/` — Обработчики событий (Consumers) из брокера сообщений RabbitMQ (`handlers.py`). Выступают эквивалентом HTTP-контроллеров для асинхронных событий.
* `dependencies.py` — Конфигурация Inversion of Control (IoC). Функции-фабрики, отвечающие за сборку графа зависимостей (Dependency Injection) для передачи готовых Use Cases в контроллеры.

## Ограничения и зависимости
* **РАЗРЕШЕНЫ** вызовы сценариев из слоя `application`.
* **РАЗРЕШЕНО** обращение к слою `infrastructure` исключительно в рамках файла `dependencies.py` для сборки DI-контейнера.
* **СТРОГО ЗАПРЕЩЕНА** любая бизнес-логика. Контроллеры не принимают решений (например, проверка баланса перед оплатой недопустима на уровне роутера).
* **СТРОГО ЗАПРЕЩЕНЫ** прямые SQL-запросы или работа с ORM-моделями в обход слоя Application.
