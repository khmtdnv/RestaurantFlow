# Infrastructure

Слой содержит технические адаптеры для взаимодействия с внешними системами (базы данных, брокеры сообщений, внешние API). Он реализует интерфейсы (контракты), заданные в слое Domain.

## Основное правило
Слой инфраструктуры является заменяемым. Изменение базы данных (например, переход с PostgreSQL на MongoDB) или брокера сообщений должно требовать изменений только в этом слое, не затрагивая бизнес-логику приложения.

## Структура директорий
* `/database/` — Компоненты для работы с реляционной базой данных (SQLAlchemy).
  * `/models/` — Декларативные ORM-модели (схемы таблиц).
  * `/migrations/` — Конфигурация и скрипты миграций Alembic.
  * `session.py` — Управление жизненным циклом соединений с БД.
* `/redis/` — Управление пулами соединений к Redis-кластеру.
* `/rabbitmq/` — Настройки подключений, обменников (Exchanges) и очередей (Queues) RabbitMQ.
* `/repositories/` — Конкретные реализации паттерна Repository для доступа к данным (ORM-адаптеры).
* `/uow/` — Физическая реализация паттерна Unit Of Work (управление транзакциями БД).
* `/dependencies/` — Провайдеры (генераторы) низкоуровневых соединений для использования в DI-контейнере.

## Ограничения и зависимости
* **РАЗРЕШЕНЫ** импорты из слоев `domain` (для реализации интерфейсов) и `application` (для использования DTO, если это необходимо при маппинге).
* **СТРОГО ЗАПРЕЩЕНО** размещение бизнес-логики. Инфраструктурный код отвечает исключительно за транспорт, хранение и трансформацию данных (CRUD).
* **СТРОГО ЗАПРЕЩЕНЫ** импорты из слоя `presentation`.
